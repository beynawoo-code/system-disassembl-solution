切换角色后顶栏主题色、子系统菜单、面包屑 URL 全部联动 — 这是登录后用户实际看到的界面雏形

多权限角色策略（登录直达 + 顶栏可切换）
默认落地：按 `primaryPlatform + primarySystem` 直达；如未配置则走 `/home`。
平台切换：顶栏右侧第一个下拉展示 `visiblePlatforms`，可在已授权平台间自由切换。
子系统切换：顶栏右侧第二个下拉展示当前平台的 `visibleSystems[platform]`。
权限边界：未授权平台/子系统不展示或灰显，后端 `scope` 强校验。
高级管理员示例：默认 `/support/users/dashboard`，可切换 `ops/delivery/quality/user/ai/project/cognition`。
CTO 示例：默认 `/ops/omnichannel/cockpit/overview`，可切换 `user/delivery/quality/project/ai/support` 下已授权子系统。
3
登录后到底跳哪儿 · 落地决策链
优先级链：深链 → 主岗位 → 唯一可见平台 → lastVisited → /home。任一无权限 → 403。
1
URL 含 deepLink？例 /ops/ads/dashboard
是 →校验平台 / 子系统 / scope，全部通过则直达；任一失败 → 403
否 →进入下一判断
2
是否有可见平台？
无 →"暂无可访问平台"页 + 引导联系管理员
有 →进入下一判断
3
user.primaryPlatform 仍可见？
是 →落地"主平台 / 主子系统"（主子系统失效则落地主平台默认 + Toast 提示）
否 →进入下一判断
4
仅 1 个可见平台？
是 →直达该平台默认子系统
否 →进入下一判断
5
lastVisited 仍可见？
是 →恢复上次离开的子系统页
否 →落地 /home 个人工作台（待办 + 平台卡片 + 收藏）
4
URL 命名规范 · 三段式
所有子系统按统一语法收拢，鉴权、埋点、深链、面包屑均以此为锚。
/{platform}/{system}/{module}/{resource}
/ops/ads/dashboard · 广告整体看板
/ops/sales/orders/:orderId · 订单详情
/delivery/procurement/pos/:poId · 采购单详情（交付平台）
/delivery/logistics/shipments · 物流发运
/quality/iqc/batches/:id · 来料检验批次
/user/customer-service/tickets/:id · 客服工单详情
/support/finance/vouchers/:id · 凭证详情
/support/contract/templates · 合同模板
/project/pct/projects/:id · PCT 项目
/ops/omnichannel/cockpit/overview · 智能驾驶舱总览（OC-GTM）
/ops/omnichannel/cockpit/profit · 利润瀑布图
/ai/workflow/builder/:flowId · 工作流编辑器
/cognition/app/posts/:postId · 认知文章详情
/cognition/admin/review · 内容审核工作台
/home · 个人工作台（多平台用户兜底）

Scope 命名同步规则：{platform}:{system}:{action} · 例 ops:ads:read / user:cs:ticket-reply / delivery:procurement:approve-po / quality:iqc:release / support:finance:close-period

5
运营保持单纯 · 三层归位法
"运营"不是一个系统，它是一种行为。拆三层归位，菜单上不再出现"运营系统"这个词。
L1 · 项目化

有起止的运营动作

跨域协同 · 项目化管理
归属 → /project
新品上市PCT 项目大促 Campaign退市清库
L2 · 例行性

每日 / 每周的执行动作

谁的域，进谁的工作台
归属 → 各子系统内部
调价 → ops/sales广告调优 → ops/ads客诉 → user/customer-service补货 → delivery/procurement
L3 · 经营性

看数与决策

横向看板 · OC-GTM 操盘
归属 → /ops/omnichannel 智能驾驶舱
BP 达成毛利预警库存周转Baseline 偏差
6
关键决策清单（D1 – D6）
本次评审需逐项确认；任一被否决则方案升版。
D1

顶栏切换器形态

两级切换器：左"平台"、右"子系统"
替代当前 18 项扁平下拉；平台稳定 8 项，子系统按平台动态加载。
D2

URL 命名

/{platform}/{system}/{module}/{resource}
鉴权 / 埋点 / 面包屑 / 深链分享所需的最小语法。
D3

登录落地策略

深链 → 主岗位 → 唯一可见平台 → lastVisited → /home
让一线零选择直达，新员工 / 多岗位有引导。
D4

站外广告归属

收编进 ops/ads 内部 tab，不单立
与已定口径一致；避免维度错位。
D5

运营纯化

不立"运营系统"；运营行为按三层下沉
防止运营变成大杂烩；详见 §5。
D6

鉴权模型

RBAC + Scope；前端软可见、后端强校验
前端永远不能成为权限的最后一道闸。
